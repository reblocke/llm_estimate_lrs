# Large Language Models and Diagnostic Likelihood Ratios

Code and curated workbooks for the manuscript:

**Large Language Models Generate Diagnostic Likelihood Ratios with Low Mean Bias but Wide Dispersion**

This project evaluates likelihood-ratio (LR) outputs generated by large language models (LLMs) against literature-reported diagnostic LRs curated from [TheNNT.com](https://thennt.com/home-lr/). The analyses focus on agreement with reported values, calibration behavior, and supplemental sensitivity analyses.

## What's Here

- `data_analysis.ipynb` — main manuscript analyses, figures, Bland-Altman agreement summaries, qualitative agreement tables, and inverse reliability-calibration outputs.
- `additional_requested_analyses.ipynb` — additional non-API analyses, including reliability zones, shortcut baselines, true LR+ versus LR- stratification, posterior-impact summaries, and a descriptive GPT-5 laboratory discrepancy table.
- `threshold_perturbation_sensitivity_analysis.ipynb` — exploratory GPT-5 threshold-perturbation analysis. Running this notebook makes OpenAI API calls.
- `lr_scraper_estimator.ipynb` — original workflow for scraping TheNNT LR tables, parsing reported LRs, querying models, and collating workbooks. Network and API-writing steps are disabled by default with explicit `RUN_*` flags.
- `NNT_LRs_08-26-2025.xlsx` — manually validated analysis workbook used by the manuscript analyses.
- `nnt_lrs_with_estimated.xlsx` — model-generated LR columns and condition-sheet labels used by the analysis notebooks.
- `pyproject.toml` and `uv.lock` — pinned Python environment.
- `.env.example` — required environment variable name for API runs.
- `LICENSE` — MIT license.

## Quick Start

Install dependencies with `uv`:

```bash
pip install uv
uv sync
uv run jupyter lab
```

To run notebooks that query the OpenAI API, create a local `.env` file:

```ini
OPENAI_API_KEY=sk-...
```

The main analysis notebooks can be executed without API calls because they use the included workbooks:

```bash
uv run jupyter nbconvert --to notebook --execute data_analysis.ipynb \
  --output tmp/data_analysis_executed.ipynb

uv run jupyter nbconvert --to notebook --execute additional_requested_analyses.ipynb \
  --output tmp/additional_requested_analyses_executed.ipynb
```

Generated outputs are written to dated folders under `results/`.

## Data and Workbooks

The analysis workbook contains diagnostic findings, literature-reported LRs, feature-type labels, and model-generated LR outputs. The source diagnostic LR values were curated from TheNNT LR pages and manually reconciled where source text required clinical context, such as missing assay thresholds or page-title information.

Fresh model-query runs may differ from the included workbook because API models and sampling behavior can change over time. Treat any rerun of `lr_scraper_estimator.ipynb` or `threshold_perturbation_sensitivity_analysis.ipynb` as a new model run.

## Methods Notes

- Agreement is summarized primarily on the natural-log LR scale.
- Bland-Altman analyses report multiplicative bias and limits of agreement on the reported/model ratio scale.
- Calibration and reliability analyses distinguish model-generated LR behavior conditional on literature-reported values from inverse reliability calibration, which asks what reported LR is expected given a model-generated LR.
- Supplemental analyses include reliability zones, heuristic shortcut baselines, posterior-impact summaries, laboratory discrepancy descriptions, and threshold perturbation checks.
