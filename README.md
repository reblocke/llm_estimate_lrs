# Large Language Models and Diagnostic Likelihood Ratios

This repository contains the analysis code and curated workbooks for the manuscript:

**Large Language Models Generate Diagnostic Likelihood Ratios with Low Mean Bias but Wide Dispersion**

The study compares likelihood-ratio (LR) outputs generated by large language models (LLMs) with literature-reported diagnostic LRs curated from TheNNT.com. The repository is intended to support reproducibility of the manuscript analyses and reviewer-requested sensitivity analyses. Generated manuscript figures, tables, draft documents, scratch notebooks, and historical working files are not tracked in the source release.

### Notebooks

- `data_analysis.ipynb` reproduces the main manuscript analyses, figures, Bland-Altman agreement summaries, qualitative agreement tables, and inverse reliability-calibration outputs from the prepared workbook.
- `reviewer1_followup_analyses.ipynb` reproduces reviewer follow-up analyses that do not require new model calls, including reliability zones, shortcut baselines, true LR+ versus LR- stratification, posterior-impact summaries, and the descriptive GPT-5 laboratory discrepancy table.
- `reviewer2_threshold_perturbation_analysis.ipynb` is a self-contained exploratory GPT-5 threshold-perturbation analysis. Re-running it will make new OpenAI API calls.
- `lr_scraper_estimator.ipynb` documents the original scraping, LR parsing, model-querying, and workbook-collation workflow. Network and API-writing steps are disabled by default with explicit `RUN_*` flags. Re-running model-query cells requires an OpenAI API key and may produce model-version or sampling differences.

### Workbooks

- `NNT_LRs_08-26-2025.xlsx` is the manually validated analysis workbook used by the manuscript analyses.
- `nnt_lrs_with_estimated.xlsx` contains model-generated LR columns and condition-sheet labels used by the analysis and reviewer follow-up notebooks.

Historical intermediate workbooks, generated retry-audit CSVs, dated result folders, and prior manuscript drafts used during project development are intentionally excluded from the release repository. The estimator notebook can recreate intermediate workbooks when its guarded scrape/query/conversion flags are intentionally enabled.

## Environment Setup

This project uses `uv` with a pinned lockfile.

```bash
pip install uv
uv sync
uv run jupyter lab
```

To run notebooks that query the OpenAI API, create a local `.env` file:

```ini
OPENAI_API_KEY=sk-...
```

Do not commit `.env`. The tracked `.env.example` file documents the required variable name only.

## Reproducing Analyses

The main manuscript and reviewer follow-up notebooks can be executed without API calls because they use the included workbooks.

```bash
uv run jupyter nbconvert --to notebook --execute data_analysis.ipynb \
  --output tmp/data_analysis_executed.ipynb

uv run jupyter nbconvert --to notebook --execute reviewer1_followup_analyses.ipynb \
  --output tmp/reviewer1_followup_executed.ipynb
```

The threshold perturbation notebook calls the OpenAI API when executed. The estimator notebook keeps live scraping, model-querying, and workbook-conversion steps behind `RUN_SCRAPE`, `RUN_OPENAI_BATCH`, and `RUN_CONVERT_TO_MASTER` flags. Treat fresh API outputs as new runs rather than exact regeneration of the manuscript workbook.

Generated outputs are written to dated folders under `results/` or `Results/`, depending on notebook compatibility logic. These folders are ignored by Git because they are derived artifacts.

## Reviewer-Requested Analyses

The current release includes code for the reviewer-requested additions:

- inverse reliability calibration with reported log-LR as outcome and model-generated log-LR as predictor;
- central versus extreme reliability-zone summaries;
- true LR+ versus LR- stratified calibration and shortcut-baseline analyses;
- exploratory GPT-5 threshold perturbation analysis;
- descriptive GPT-5 laboratory discrepancy table based on curated finding text.

These analyses are framed as supplemental sensitivity or descriptive analyses. They do not prove internal model reasoning and do not validate use of LLM-generated LRs for unstudied clinical findings.

## Release Hygiene

The source release intentionally excludes:

- API keys and local environment files;
- virtual environments and caches;
- dated result folders and generated manuscript outputs;
- historical manuscript drafts and response-letter drafts;
- scratch executed notebooks and rendered audit images;
- historical past-run workbooks and old zip archives.

Use the notebooks and included workbooks as the reproducible source of truth. Use generated outputs from a dated run only as derived artifacts.
